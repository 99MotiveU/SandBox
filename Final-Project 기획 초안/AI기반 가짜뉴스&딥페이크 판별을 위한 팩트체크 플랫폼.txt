AI 기반 가짜뉴스/딥페이크 판별을 위한 팩트체크 플랫폼

1. 사회 문제 정의
생성형 AI 기술의 발전으로 진짜와 구별하기 어려운 가짜뉴스, 딥페이크 영상/음성이 무분별하게 유포되고 있습니다. 이는 사회적 불신을 증폭시키고 선거 등 민주주의 근간을 위협하며, 보이스피싱과 같은 범죄에도 악용되어 심각한 피해를 낳고 있습니다. 일반 시민들은 미디어 콘텐츠의 진위를 판별할 수단이 없어 혼란에 빠지기 쉽습니다.

2. 기획 의도 및 해결 방안
시민들이 의심스러운 뉴스 기사나 이미지를 손쉽게 제보하고, **AI 기술과 집단지성(크라우드소싱)**을 결합하여 진위를 판별하고 그 결과를 투명하게 공유하는 팩트체크 플랫폼을 구축합니다.

3. 핵심 기능
의심 콘텐츠 제보 및 분석 요청:
사용자가 의심스러운 뉴스 기사의 URL, 텍스트, 이미지 파일을 업로드하여 분석을 요청.
AI 기반 1차 분석:
(텍스트): 제보된 기사 텍스트를 외부 LLM(Large Language Model) API (e.g., Google Gemini, OpenAI GPT)에 전송하여 "편향성 분석", "자극적 표현 탐지", "사실관계 오류 가능성" 등을 1차적으로 분석한 결과 제시.
(이미지): 외부 딥페이크 탐지 API를 연동하여 이미지의 위변조 가능성을 퍼센트로 표시.
전문가/시민 팩트체커 시스템:
언론인, 교수 등 사전에 검증된 '전문 팩트체커' 그룹과, 일정 기준을 통과한 '시민 팩트체커' 그룹을 운영.
제보된 콘텐츠에 대해 팩트체커들이 교차 검증을 진행하고, 근거 자료와 함께 '사실', '대체로 거짓', '판단 보류' 등의 판정 결과를 등록.
판별 결과 아카이빙 및 검색:
지금까지 판별이 완료된 모든 콘텐츠를 데이터베이스에 저장하고, 키워드로 쉽게 검색할 수 있도록 제공.
4. 기술 스택 및 포인트
Backend: Spring Boot, WebClient (외부 AI API 비동기 호출)
AI/ML: 외부 API 연동에 집중. (직접 모델을 만드는 것은 범위 초과)
Database: MySQL (구조화된 데이터), Elasticsearch (텍스트 검색 기능 강화)
Key Points:
외부 AI API 연동: Spring WebClient를 활용하여 외부의 복잡한 AI 서비스를 내 애플리케이션의 기능으로 통합하는 실전 경험.
비동기 처리: AI 분석에 시간이 걸리는 점을 감안하여, 분석 요청을 비동기적으로 처리하고 완료 시 사용자에게 알림을 주는 아키텍처 설계 (@Async, RabbitMQ 등).
평판/신뢰도 시스템: 팩트체커의 활동 내역을 기반으로 신뢰도 점수를 부여하는 시스템 로직 구현.
5. 기대 효과 및 포트폴리오 가치
AI 시대의 가장 뜨거운 사회 문제인 '정보의 신뢰성'을 다루는 프로젝트.
단순 웹 개발을 넘어 외부 AI 서비스를 활용하고, 비동기 처리, 검색 엔진 등 고도화된 기술을 적용해보는 경험을 통해 차별화된 기술 역량을 보여줄 수 있음.
