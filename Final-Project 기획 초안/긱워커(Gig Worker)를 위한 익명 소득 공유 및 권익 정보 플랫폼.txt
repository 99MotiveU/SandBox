긱워커(Gig Worker)를 위한 익명 소득 공유 및 권익 정보 플랫폼

1. 사회 문제 정의
배달 라이더, 대리운전 기사, 크리에이터, 프리랜서 디자이너 등 플랫폼을 기반으로 일하는 '긱워커'가 급증하고 있습니다. 이들은 전통적인 근로자와 달리 소득이 불안정하고, 플랫폼의 정책(수수료, 평점 시스템 등)에 일방적으로 끌려가며, 세금 신고나 보험 등에서 정보 부족으로 어려움을 겪는 경우가 많습니다. 자신의 소득이 업계 평균에 비해 적정한지, 다른 플랫폼의 조건은 어떤지 객관적으로 비교할 정보 채널이 없습니다.

2. 기획 의도 및 해결 방안
긱워커들이 서로의 소득과 근무 조건을 익명으로 공유하고, 플랫폼별 장단점을 투명하게 평가하며, 세무/법률/금융 등 필수 정보를 얻을 수 있는 긱워커 전용 커뮤니티 및 데이터 플랫폼을 구축합니다.

3. 핵심 기능
익명 소득 인증 및 데이터 공유:
사용자가 플랫폼 앱의 소득 정산 내역을 캡처해서 올리면, OCR 기술로 텍스트를 자동 추출하여 근무 시간, 플랫폼, 수입 등을 구조화된 데이터로 저장 (개인정보는 마스킹).
인증된 사용자만 다른 사람의 소득 정보를 열람할 수 있도록 제한.
플랫폼별 비교/분석 대시보드:
축적된 데이터를 바탕으로 플랫폼별, 지역별, 경력별 평균 시급/월급 통계를 시각화하여 제공.
"A플랫폼은 수수료는 높지만 콜이 많고, B플랫폼은 단가는 낮지만 프로모션이 강력함" 등 실제 데이터를 기반으로 한 비교 분석 리포트 제공.
'그들만의 리그' 익명 커뮤니티:
플랫폼별 게시판을 두어 불합리한 정책에 대한 공론화, 꿀팁 공유, 고충 토로 등을 자유롭게 할 수 있는 공간 마련.
긱워커 맞춤 정보 큐레이션:
종합소득세 신고 방법, 고용보험/산재보험 가입 안내, 긱워커를 위한 저금리 대출 상품 정보 등 흩어져 있는 필수 정보들을 모아서 제공.
4. 기술 스택 및 포인트
Backend: Spring Boot, Spring Security (익명성 보장 로직 중요)
OCR: Naver Clova OCR, Google Vision API 등 외부 API 활용.
Database: MySQL 또는 MariaDB. 통계 처리를 위한 QueryDSL 적극 활용.
Key Points:
데이터 추출 및 정제: 비정형 데이터(이미지)에서 OCR로 텍스트를 추출하고, 유의미한 정보(수입, 시간 등)를 파싱하여 정형 데이터로 가공하는 파이프라인 구축 경험.
통계 및 데이터 시각화: 복잡한 조건의 통계 데이터를 DB에서 효율적으로 집계하고, 차트 라이브러리와 연동하여 사용자에게 직관적으로 보여주는 능력.
개인정보보호 및 익명성: 사용자의 민감 정보를 안전하게 처리하고, 익명성을 보장하기 위한 아키텍처 설계 고민.
5. 기대 효과 및 포트폴리오 가치
새로운 노동 패러다임의 문제점을 파고드는 시의성 높은 프로젝트.
데이터 기반의 정보 비대칭 해소라는 명확한 가치를 제공하며, 데이터 수집-가공-분석-시각화의 전체 사이클을 경험한 개발자임을 증명.
